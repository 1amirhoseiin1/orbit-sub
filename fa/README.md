# پنل اشتراک (Subscription Panel)

پنل تک‌صفحه‌ای برای نمایش وضعیت اشتراک یک کاربر: نام و وضعیت، تاریخ انقضا، نمودار
دایره‌ای مصرف حجم، لینک اشتراک قابل اشتراک‌گذاری با QR، و لیست کانفیگ‌ها با
دکمه کپی و QR جداگانه برای هرکدام.

## پشته فنی

- **React 18 + Vite** برای dev server سریع و build بهینه
- **MUI (Material UI)** برای کامپوننت‌های آماده و در دسترس (Avatar, Chip,
  CircularProgress, Popover, Tooltip, IconButton)
- **Tailwind CSS** برای چیدمان، فاصله‌گذاری، افکت‌های شیشه‌ای و انیمیشن پس‌زمینه
- **qrcode.react** برای تولید QR کاملاً سمت کلاینت (بدون فراخوانی API خارجی)
- **gsap + @gsap/react** برای جلوه‌ی حرکتی تیتر (`SplitText.jsx`)

### چرا هم MUI هم Tailwind؟

MUI مسئول کامپوننت‌های تعاملی و دسترس‌پذیر است (فوکوس، ARIA، ریپل و غیره) —
کارهایی که بازسازی‌شان با کلاس‌های Tailwind وقت‌گیر و پرخطاست. Tailwind مسئول
لایه بصری خودمان است: کارت‌های شیشه‌ای، گرید، فاصله‌ها، انیمیشن پس‌زمینه.

برای اینکه این دو با هم تداخل نکنند:

- `preflight` تیلویند در `tailwind.config.js` خاموش شده، چون MUI با
  `CssBaseline` ریست خودش را اعمال می‌کند.
- رنگ‌ها در دو جا تعریف شده‌اند (`theme.js` برای MUI و `tailwind.config.js`
  برای Tailwind) اما مقادیر هگز یکسان هستند - در واقع یک توکن رنگی مشترک با دو
  نحو مختلف.
- کامپوننت‌های MUI با پراپ `sx` استایل می‌گیرند، بقیه‌ی لایه‌بندی با کلاس‌های
  Tailwind. این تفکیک را در کل پروژه رعایت کنید تا کد قابل پیش‌بینی بماند.

## اجرای محلی

```bash
npm install
npm run dev
```

سپس آدرس نمایش‌داده‌شده در ترمینال (پیش‌فرض `http://localhost:5173`) را باز کنید.

## Build و Deploy

```bash
npm run build      # خروجی در پوشه dist/
npm run preview    # پیش‌نمایش build نهایی
```

با `vite-plugin-singlefile` که در `vite.config.js` اضافه شده، خروجی build فقط
یک فایل `dist/index.html` است — CSS و JS به‌جای پوشه‌ی `assets/` مستقیم داخل
همان HTML inline می‌شوند. یعنی می‌توانید همان یک فایل را جایی که خواستید (حتی
بدون هیچ static host ای، با باز کردن مستقیم در مرورگر) استفاده کنید.

پوشه `dist/` (یا فقط همان یک فایل `index.html`) را می‌توانید روی Vercel،
Netlify، Cloudflare Pages یا هر static host دیگری هم آپلود کنید. نیازی به سرور
Node در پروداکشن نیست.

## اتصال به API واقعی

فعلاً `src/App.jsx` داده‌ها را از `src/data/mockData.js` می‌خواند. برای اتصال
به بک‌اند واقعی:

```jsx
// در App.jsx، به‌جای import از mockData:
const [data, setData] = useState(null);
useEffect(() => {
  fetch('/api/subscription').then((r) => r.json()).then(setData);
}, []);
```

شکل داده باید با همان سه شیء `user` / `subscription` / `configs` مطابقت داشته
باشد که در `mockData.js` تعریف شده‌اند.

## نکته‌ی مسیریابی (Routing)

این پنل به‌صورت تک‌صفحه طراحی شده و به router نیازی ندارد. اگر بعداً صفحات
بیشتری اضافه شد (مثلاً صفحه ورود یا تاریخچه پرداخت‌ها)، `react-router-dom` را
اضافه کنید و `SubscriptionPanel` را زیر یک مسیر مثل `/panel/:userId` قرار
دهید؛ چون کامپوننت از قبل props را از بیرون می‌گیرد، نیازی به تغییر داخل آن
نیست.

## اتصال به مرزبان (Marzban)

این پنل طوری طراحی شده که مستقیماً به‌عنوان **تمپلیت صفحه‌ی اشتراک مرزبان**
(`SUBSCRIPTION_PAGE_TEMPLATE`) قابل استفاده باشه. نکته‌ی مهم اینه که مرزبان
داده‌ی کاربر رو با موتور Jinja2 **سمت سرور** داخل HTML رندر می‌کنه (نه با
فراخوانی API از سمت مرورگر) - دقیقاً همون کاری که تمپلیت پیش‌فرض خودش
(`app/templates/subscription/index.html`) با چیزهایی مثل `{{ user.username }}`
و `{% for link in user.links %}` انجام می‌ده.

برای همین، داخل `index.html` یک بلوک با `id="marzban-data"` و
`type="application/json"` گذاشته شده که همین سینتکس Jinja رو داره. مرزبان وقتی
این فایل رو serve می‌کنه، مقادیر واقعی رو جایگزین می‌کنه؛ اپ React ما هم موقع
اجرا این بلوک رو می‌خونه (`src/data/marzbanAdapter.js`). اگه این بلوک خالی یا
رندر‌نشده باشه (مثلاً موقع `npm run dev` روی سیستم خودت)، به‌صورت خودکار از
داده‌ی mock استفاده می‌شه - یعنی نیازی به هیچ سرور یا CORS برای توسعه‌ی لوکال
نیست.

### راه‌اندازی روی سرور مرزبان

```bash
# ۱. build بگیر (خروجی یک فایل تک HTML است)
npm install
npm run build

# ۲. فایل build شده رو به پوشه‌ی تمپلیت مرزبان کپی کن
sudo mkdir -p /var/lib/marzban/templates/subscription
sudo cp dist/index.html /var/lib/marzban/templates/subscription/index.html

# ۳. این دو متغیر رو به /opt/marzban/.env اضافه کن (اگر قبلاً نبود)
echo 'CUSTOM_TEMPLATES_DIRECTORY="/var/lib/marzban/templates/"' | sudo tee -a /opt/marzban/.env
echo 'SUBSCRIPTION_PAGE_TEMPLATE="subscription/index.html"' | sudo tee -a /opt/marzban/.env

# ۴. مرزبان رو ری‌استارت کن
marzban restart
```

بعد از این مراحل، لینک اشتراک هر کاربر (`.../sub/<token>`) وقتی توی مرورگر باز
بشه، همین پنل رو با نام کاربر، وضعیت، حجم مصرفی و لیست کانفیگ‌های واقعی همون
کاربر نشون می‌ده.

### فیلدهایی که از مرزبان خونده می‌شن

| فیلد Marzban (`user.*`)   | استفاده در پنل                          |
| ------------------------- | ---------------------------------------- |
| `username`                | نام نمایشی بالای پنل                     |
| `status`                  | فعال/غیرفعال (سبز/قرمز)                  |
| `data_limit`              | حجم کل (اگر خالی باشه یعنی نامحدود)      |
| `used_traffic`            | حجم مصرف‌شده                              |
| `expire`                  | تاریخ انقضا (اگر خالی باشه یعنی نامحدود) |
| `subscription_url`        | لینک بالای QR اشتراک                     |
| `links`                   | لیست کانفیگ‌ها (VLESS/VMess/Trojan/...)   |

### نکات مهم / محدودیت‌ها

- **SplitText و فارسی:** `SplitText.jsx` به‌صورت پیش‌فرض متن رو بر اساس کلمه
  (`splitType="words"`) می‌شکنه، نه حرف. حروف فارسی/عربی به هم می‌چسبن (خط
  کرسیو)؛ اگه حرف‌به‌حرف بشکنی، این اتصال از بین می‌ره و متن زشت می‌شه. برای
  متن انگلیسی/لاتین می‌تونی `splitType="chars"` بدی، ولی برای متن فارسی همیشه
  `words` رو نگه دار.
- **ایموجی پرچم روی ویندوز:** ویندوز پرچم کشورها رو به‌صورت آیکون رسم نمی‌کنه
  (به‌جاش حروف کشور رو نشون می‌ده). برای رفع این مشکل از **Twemoji** (ست آیکون
  متن‌باز توییتر) توی `src/components/Emoji.jsx` استفاده شده که پرچم‌ها رو با
  یک SVG کوچیک از CDN جایگزین می‌کنه - دقیقاً مثل فونت گوگل که همین الان هم
  استفاده می‌کنیم، این یعنی کاربر برای دیدن پرچم‌ها به اینترنت نیاز داره.
- **لینک اشتراک نسبی:** بعضی نصب‌های مرزبان `user.subscription_url` رو به‌صورت
  مسیر نسبی (`/sub/...`) برمی‌گردونن نه آدرس کامل. `marzbanAdapter.js` این رو
  خودکار با `window.location.origin` کامل می‌کنه.
- **آیکون‌های واقعی اپ‌ها:** توی `src/assets/app-icons/` پنج فایل هست
  (`streisand.png`, `v2rayng.png`, `v2box.png`, `v2raytun.png`, `happ.png`).
  این تطبیق بر اساس حدس از روی شکل هر آیکون انجام شده (مثلاً حرف "H" حتماً
  Happ، شکل "V2" + مکعب حتماً V2Box)؛ اگه یکی‌شون جابه‌جا بود، فقط اسم فایل‌ها
  رو swap کن - جایی دیگه‌ای نیازی به تغییر نیست. شِمای هر اپ از مستندات یا
  changelog خودشون تأیید شده (v2rayNG از GitHub Issues خودش، V2Box از
  changelog اپ‌استور، V2RayTun از `docs.v2raytun.com/deep-link`، Happ از
  تمپلیت‌های رایج ساب مرزبان). **Streisand** رو هم به همین شکل
  (`streisand://import/{link}#{name}`) وصل کردیم، چون از همون الگوی
  اپ‌های هم‌خانواده‌ش پیروی می‌کنه، هرچند رسماً تو سایت خودشون تبلیغ نشده. برای
  هر اپ، کپی کردن لینک در کلیپ‌بورد به‌عنوان fallback همیشه انجام می‌شه، برای
  مواقعی که یه نسخه‌ی خاص از اپ deep link رو نگیره. چون این deep linkها به
  نصب واقعی هر اپ روی موبایل نیاز دارن، حتماً روی گوشی واقعی تستشون کن، نه
  فقط توی مرورگر دسکتاپ.
- **پرچم‌ها حدسی هستن.** مرزبان به خودیِ خود پرچم کشور رو به کانفیگ‌ها نمی‌ده؛
  `marzbanAdapter.js` از روی متن remark (بخش بعد از `#` در هر لینک کانفیگ) حدس
  می‌زنه. اگه remark از قبل ایموجی پرچم داشته باشه (روش رایج بین ادمین‌های
  مرزبان)، همون استفاده می‌شه؛ در غیر این صورت با یک دیکشنری کوچیک از اسم/کد
  کشورها تطبیق داده می‌شه. برای دقت بیشتر، لیست `COUNTRY_FLAGS` رو در همون
  فایل ویرایش کن یا مستقیم توی نام هاست‌های مرزبان پرچم بذار.
- **واحد حجم:** تبدیل بایت به گیگابایت با `1024³` انجام می‌شه (GiB). اگه
  می‌خوای دقیقاً با همون عددی که مرزبان با فیلتر `bytesformat` نشون می‌ده یکی
  باشه، همین مقدار `BYTES_PER_GB` رو در `marzbanAdapter.js` چک کن.
- **کاراکترهای خاص در remark:** چون بلوک داده داخل `<script type="application/json">`
  با Jinja پر می‌شه، اگر نام یک کاربر یا remark یک کانفیگ حاوی کوتیشن دوتایی
  (`"`) باشه، ممکنه JSON خراب بشه. برای اکثر کاربردها (اسم کاربر، remark
  سرورها) این اتفاق نمی‌افته، ولی اگر روی این مورد حساسیت داری بهم بگو تا یک
  لایه escape قوی‌تر اضافه کنم.



- تمام دکمه‌ها `aria-label` دارند و با `Tab` قابل دسترسی‌اند.
- فوکوس کیبورد یک outline بنفش مشخص دارد (`:focus-visible` در `index.css`).
- کنتراست متن روی پس‌زمینه تیره مطابق حداقل WCAG AA رعایت شده (متن اصلی
  `#FAFAFA` روی `#09090B`/`#18181B`).
- انیمیشن‌ها با `prefers-reduced-motion: reduce` غیرفعال می‌شوند.

## فازبندی پیشنهادی

**فاز ۱ - MVP**
- چیدمان اصلی، هدر وضعیت کاربر، نمودار مصرف ساده (بدون انیمیشن)، لینک اشتراک
  با QR، لیست کانفیگ‌ها با کپی. داده mock.

**فاز ۲ - تقویت**
- انیمیشن پرشدن نمودار دایره‌ای، پس‌زمینه‌ی متحرک، افکت glow روی هاور کارت‌ها،
  ریزتنظیمات ریسپانسیو برای موبایل، بررسی دسترس‌پذیری (فوکوس، کنتراست،
  reduced motion)، و در نهایت اتصال به API واقعی به‌جای mock data.
