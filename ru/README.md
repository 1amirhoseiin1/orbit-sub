# Панель подписки (Subscription Panel)

Одностраничная панель для отображения статуса подписки пользователя:
имя и статус, дата окончания, круговая диаграмма расхода трафика,
ссылка на подписку с QR-кодом, и список конфигураций с кнопкой
копирования и отдельным QR для каждой.

## Стек технологий

- **React 18 + Vite** для быстрого dev-сервера и оптимизированной сборки
- **MUI (Material UI)** для готовых и доступных компонентов (Avatar, Chip,
  CircularProgress, Popover, Tooltip, IconButton)
- **Tailwind CSS** для вёрстки, отступов, стеклянных эффектов и анимации фона
- **qrcode.react** для генерации QR полностью на клиенте (без внешних API)
- **gsap + @gsap/react** для анимации заголовка (`SplitText.jsx`)

### Почему одновременно MUI и Tailwind?

MUI отвечает за интерактивные и доступные компоненты (фокус, ARIA, ripple
и т.д.) - то, что дорого и рискованно пересобирать вручную на классах
Tailwind. Tailwind отвечает за наш собственный визуальный слой: стеклянные
карточки, сетку, отступы, анимацию фона.

Чтобы эти два инструмента не конфликтовали:

- `preflight` Tailwind отключён в `tailwind.config.js`, потому что MUI
  сам применяет свой сброс через `CssBaseline`.
- Цвета определены в двух местах (`theme.js` для MUI и
  `tailwind.config.js` для Tailwind), но значения HEX идентичны - по сути
  один общий цветовой токен с двумя разными синтаксисами.
- Компоненты MUI стилизуются через проп `sx`, остальная вёрстка - классами
  Tailwind. Соблюдайте это разделение во всём проекте, чтобы код оставался
  предсказуемым.

## Локальный запуск

```bash
npm install
npm run dev
```

Затем откройте адрес, указанный в терминале (по умолчанию
`http://localhost:5173`).

## Сборка и деплой

```bash
npm run build      # результат в папке dist/
npm run preview    # предпросмотр финальной сборки
```

Благодаря `vite-plugin-singlefile` в `vite.config.js`, результат сборки -
это всего один файл `dist/index.html`: CSS и JS встроены прямо в него
вместо отдельной папки `assets/`. Это значит, что этот один файл можно
использовать где угодно, даже без static-хостинга - просто открыв его
напрямую в браузере.

Папку `dist/` (или просто файл `index.html`) можно также загрузить на
Vercel, Netlify, Cloudflare Pages или любой другой static-хостинг. Node-сервер
в продакшене не требуется.

## Подключение к реальному API

Сейчас `src/App.jsx` читает данные из `src/data/marzbanAdapter.js`, который,
в свою очередь, при отсутствии данных от Marzban (например, при локальной
разработке) откатывается на `src/data/mockData.js`. Чтобы подключить
собственный бэкенд вместо Marzban:

```jsx
// в App.jsx вместо getPanelData() из marzbanAdapter:
const [data, setData] = useState(null);
useEffect(() => {
  fetch('/api/subscription').then((r) => r.json()).then(setData);
}, []);
```

Форма данных должна соответствовать тем же трём объектам `user` /
`subscription` / `configs`, что определены в `mockData.js`.

## Заметка о роутинге

Эта панель спроектирована как одна страница и не требует роутера. Если в
будущем появятся другие страницы (например, страница входа или история
платежей), добавьте `react-router-dom` и разместите `SubscriptionPanel`
под маршрутом вроде `/panel/:userId`; поскольку компонент уже получает
все данные через props, менять что-то внутри него не понадобится.

## Подключение к Marzban

Эта панель спроектирована так, чтобы использоваться напрямую как
**шаблон страницы подписки Marzban** (`SUBSCRIPTION_PAGE_TEMPLATE`). Важный
момент: Marzban рендерит данные пользователя через движок Jinja2 **на
стороне сервера** (а не через вызов API из браузера) - точно так же, как
это делает его собственный шаблон по умолчанию
(`app/templates/subscription/index.html`) с конструкциями вроде
`{{ user.username }}` и `{% for link in user.links %}`.

Поэтому внутри `index.html` есть блок с `id="marzban-data"` и
`type="application/json"`, использующий тот же синтаксис Jinja. Когда
Marzban отдаёт этот файл, он подставляет реальные значения; наше React-
приложение при запуске читает этот блок (`src/data/marzbanAdapter.js`).
Если блок пуст или не был обработан (например, при `npm run dev` на вашей
машине), автоматически используются тестовые данные - то есть для локальной
разработки не нужен ни сервер, ни CORS.

### Установка на сервере Marzban

```bash
# 1. Собрать проект (результат - один HTML-файл)
npm install
npm run build

# 2. Скопировать собранный файл в папку шаблонов Marzban
sudo mkdir -p /var/lib/marzban/templates/subscription
sudo cp dist/index.html /var/lib/marzban/templates/subscription/index.html

# 3. Добавить эти две переменные в /opt/marzban/.env (если их ещё нет)
echo 'CUSTOM_TEMPLATES_DIRECTORY="/var/lib/marzban/templates/"' | sudo tee -a /opt/marzban/.env
echo 'SUBSCRIPTION_PAGE_TEMPLATE="subscription/index.html"' | sudo tee -a /opt/marzban/.env

# 4. Перезапустить Marzban
marzban restart
```

После этих шагов ссылка подписки каждого пользователя (`.../sub/<token>`),
открытая в браузере, будет показывать эту панель с реальным именем,
статусом, расходом трафика и списком конфигураций именно этого пользователя.

### Какие поля читаются из Marzban

| Поле Marzban (`user.*`)   | Использование в панели                    |
| -------------------------- | ------------------------------------------ |
| `username`                 | Отображаемое имя вверху панели              |
| `status`                   | Активна/неактивна (зелёный/красный)         |
| `data_limit`                | Общий объём (если пусто - безлимит)         |
| `used_traffic`              | Использованный трафик                       |
| `expire`                    | Дата окончания (если пусто - безлимит)      |
| `subscription_url`          | Ссылка над QR подписки                      |
| `links`                     | Список конфигураций (VLESS/VMess/Trojan/…)  |

### Важные замечания / ограничения

- **Иконки приложений:** в `src/assets/app-icons/` пять файлов
  (`streisand.png`, `v2rayng.png`, `v2box.png`, `v2raytun.png`, `happ.png`).
  Схемы deep link для каждого приложения подтверждены из их собственной
  документации/changelog (v2rayNG - из GitHub Issues, V2Box - из changelog
  App Store, V2RayTun - из `docs.v2raytun.com/deep-link`, Happ - из
  распространённых шаблонов страниц подписки Marzban). **Streisand**
  подключён по тому же принципу (`streisand://import/{link}#{name}`),
  так как следует той же схеме, что и родственные приложения, хотя это
  официально не задокументировано на их сайте. Для каждого приложения
  копирование ссылки в буфер обмена всегда выполняется как запасной
  вариант - на случай, если конкретная версия приложения не подхватит
  deep link. Поскольку эти deep link'и работают только при реальной
  установке приложения на телефоне, обязательно протестируйте их на
  настоящем устройстве, а не только в браузере на компьютере.
- **Относительная ссылка подписки:** некоторые установки Marzban
  возвращают `user.subscription_url` в виде относительного пути
  (`/sub/...`), а не полного адреса. `marzbanAdapter.js` автоматически
  дополняет его через `window.location.origin`.
- **Флаги стран - приблизительны.** Marzban сам по себе не передаёт флаг
  страны для конфигураций; `marzbanAdapter.js` угадывает его по тексту
  remark (часть после `#` в каждой ссылке конфигурации), проверяя как
  английские, так и русские ключевые слова. Если в remark уже есть эмодзи
  флага (частая практика среди администраторов Marzban), используется
  именно он; иначе применяется небольшой словарь названий/кодов стран. Для
  большей точности отредактируйте список `COUNTRY_FLAGS` в этом же файле,
  либо сразу добавляйте флаг в имя хоста на стороне Marzban.
- **Единицы объёма:** перевод байтов в гигабайты выполняется через `1024³`
  (GiB). Если нужно точное соответствие числу, которое Marzban показывает
  через фильтр `bytesformat`, проверьте значение `BYTES_PER_GB` в
  `marzbanAdapter.js`.
- **Специальные символы в remark:** поскольку блок данных внутри
  `<script type="application/json">` заполняется через Jinja, если имя
  пользователя или remark конфигурации содержит двойную кавычку (`"`),
  JSON может сломаться. Для большинства случаев (имена пользователей,
  remark серверов) это не происходит, но если это критично для вас,
  дайте знать - добавим более надёжное экранирование.

## Доступность

- Все кнопки имеют `aria-label` и доступны через `Tab`.
- Фокус с клавиатуры обозначен чётким фиолетовым outline
  (`:focus-visible` в `index.css`).
- Контраст текста на тёмном фоне соответствует минимум WCAG AA (основной
  текст `#FAFAFA` на `#09090B`/`#18181B`).
- Анимации отключаются при `prefers-reduced-motion: reduce`.
